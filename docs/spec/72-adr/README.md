# Architecture Decision Records

**Status:** Draft

## Purpose

Short ADRs for costly-to-reverse choices. Keep one decision per file.

## Naming

Use `NNNN-title-in-kebab-case.md` (e.g., `0001-mesa-as-abm-engine.md`).

## Template (copy into new ADR)

- **Status:** Proposed | Accepted | Superseded
- **Context:** What forced a decision?
- **Decision:** What we chose.
- **Consequences:** Positives, negatives, follow-ups.
